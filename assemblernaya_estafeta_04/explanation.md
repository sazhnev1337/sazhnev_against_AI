### Задание 1 — скалярное произведение

Цикл по C итерациям (C=3). На каждой итерации:
1. `FMOV.O3 FQA, [A]` и `FMOV.O3 FQE, [B]` — загружаем байты OFP8 из обоих векторов.
2. `FCVT.H.O3 FHC, FQA` / `FCVT.H.O3 FHD, FQE` — конвертируем в float16.
3. `FMOV.H FHB, FHC` → `FMUL.H FHB, FHD` — перемножаем.
4. `FADD.H FHA, FHB` — накапливаем в аккумулятор FHA.
5. В конце `FMOV.H [D], FHA` — сохраняем 2 байта float16 в `acc`.

### Задание 2 — матрично-векторное умножение

Два вложенных цикла. Состояние между итерациями хранится **на стеке** в трёх ячейках: `row_counter`, `mat_ptr`, `res_ptr`. Доступ через `[SP+1]`, `[SP+2]`, `[SP+3]`.

- Внешний цикл: перебирает строки (row_counter от m до 0).
- Внутренний цикл: перебирает столбцы (C = значение n из памяти `[n]`), выполняет то же что задание 1.
- После каждой строки: `FMOV.H [D], FHA`, `ADD D, 2`, `MOV [SP+2], A` (mat_ptr обновился автоматически в inner loop).

Значения m и n читаются через прямую адресацию `MOV A, [m]` / `MOV C, [n]`.

### Задание 3 — экспонента через схему Хорнера

Аппаратного `exp` нет, используется **ряд Тейлора с 9 членами** (k=0..8), вычисленный по **схеме Хорнера**:

```
e^x ≈ 1/8! → ×x + 1/7! → ×x + ... → ×x + 1
```

Коэффициенты `1/k!` для k=8..0 хранятся в памяти как float16 (DB с суффиксом `_h`). Цикл из 8 итераций: `FMUL.H FHA, FHB` (×x) → `FADD.H FHA, FHC` (+coeff). Указатель `A` сдвигается на +2 байта за итерацию (INC A × 2). Точность достаточна для float16 при умеренных |x|.
